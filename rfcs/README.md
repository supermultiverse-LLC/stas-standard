# STAS-01 Requests for Comments

This directory contains Requests for Comments (RFCs) proposing changes to
the STAS-01 interoperability specification.

An RFC is a proposal.

It becomes normative only after:

1. review;
2. acceptance;
3. incorporation into a released version of the STAS-01 specification.

---

## Purpose

RFCs provide a structured process for evolving STAS-01 while preserving
backward compatibility, interoperability and implementation
independence.

---

## RFC Lifecycle

Every RFC progresses through one of the following states:

- Draft
- In Review
- Accepted
- Rejected
- Withdrawn
- Superseded

Accepted RFCs authorize specification changes.

They do not define conformance until those changes are incorporated into
a released specification.

---

## File Naming

RFC files use the following format:
rfc-NNNN-short-title.md

Example:
rfc-0001-canonical-object-identifier.md
